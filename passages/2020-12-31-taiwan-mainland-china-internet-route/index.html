台灣地區與大陸網絡互聯情況

簡單測試一下從大陸到台灣三大 ISP (Hinet, Seednet, TFN)的去程路由，供搭建服務器參考。

三大 ISP 簡介

  • Hinet: 隸屬中華電信，是台灣最大的 ISP，與大陸互聯質量取決於 IP，一般而言固定制（商寬）優於非固定制（家寬），固定IP優於動態IP。
  • Seednet: 隸屬遠傳電信，與大陸互聯質量不錯，適合移動用戶。
  • TFN (Taiwan Fixed Net): 隸屬台灣大哥大，由於使用的人最少，所以到大陸互聯質量反而比 Hinet 優秀，適合聯通用戶。
  • 大陸到台灣路由經常繞香港，然而台港之間互聯非常暢通，因此就算繞港，只要大陸到香港不炸仍然會有優秀的表現。

路由測試

因為我只想知道用什麼中轉拉得動，所以僅針對幾個常用中轉線路簡單測一下。

Hinet

測試IP：111.255.35.243

上海聯通：經廣州聯通直連台灣（不繞港），ping 65ms-100ms

常州移動：經廣移 → 香港移動直連台灣，ping 65ms

廣州移動：經廣移 → 香港移動直連台灣，ping 30ms

上海CN2：由上海直連台灣，ping 40-50ms

Seednet

測試IP：106.104.115.1

上海聯通：繞日本 NTT → 香港 NTT 至台灣，ping 300ms+

常州移動：經廣移 → 香港 telstra 直連台灣，ping 65ms

廣州移動：經廣移 → 香港 telstra 直連台灣，ping 30ms

上海CN2：經香港電信直連台灣，ping 65ms

TFN

測試IP：175.98.244.78

上海聯通：經香港聯通直連台灣，ping 65ms

常州移動：經廣移 → 香港 pccw 至台灣，ping 300ms+

廣州移動：經廣移 → 香港 pccw 至台灣，ping 300ms+

上海CN2：經香港電信直連台灣，ping 65ms

結論

  1. 總體好用程度：上海 CN2 > 廣州移動 >= 上海聯通（常州聯通路由幾乎和上海聯通一樣，只是延遲略高）
  2. 聯通可拉 TFN 或 Hinet；Seednet 聯通繞日，不建議
  3. 移動可拉 Seednet 或 Hinet；TFN 移動在廣州移動-香港 PCCW 段會炸，不建議
  4. 上海 CN2 拉一切台灣
  5. 台灣 Hinet 雖然路由最好，然而價格最高而且晚高峰負載大，建議選擇 Seednet 或 TFN 搭配對應中轉以獲得最佳體驗
